LKH用户接口和参数说明
输入通过两个独立的文件：
1）	Problem_file
2）	Parameter_file
即 问题文件和参数文件。

问题文件的格式的TSPLIB的格式，这个文件属于中间的文件，不需要太关心。
参数文件的格式

所有参数采用<keyword> = <value> 的格式给出具体值。<Keyword>是字母数字的关键词，<value>表示字母或数字的值。关键词不区分大小写。

强制参数:
PROBLEM_FILE = <string>
问题文件的文件名。
TOUR_FILE = <string>
输出路径文件的文件名。

可选参数：
ASCENT_CANDIDATES = <integer>
默认：50

BACKBONE_TRIALS = <integer>
默认： 0

BACKTRACKING = { YES | NO }
默认:NO

CANDIDATE_FILE = <string>
默认：无

CANDIDATE_SET_TYPE = { ALPHA | DELAUNAY [ PURE ] | NEAREST-NEIGHBOR | QUADRANT }
默认：ALPHA

COMMENT <string>
# <string>
评论或注释

EOF
输入数据的终止符。

EXCESS = <real>
默认：1.0/DIMENSION.

EXTRA_CANDIDATES = <integer> [ SYMMETRIC ]
默认：0

EXTRA_CANDIDATE_SET_TYPE = { NEAREST-NEIGHBOR | QUADRANT }

GAIN23 = [ YES | NO ]

GAIN_CRITERION ={ YES | NO }
